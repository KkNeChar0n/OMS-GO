财务管理

需求目标

提供完整的财务收支管理能力，包括常规收款、淘宝收款、待认领款项处理、分账明细查看、退费管理等核心财务功能。实现从收款到分账到退费的完整财务闭环。

  • 业务目标：确保每笔收款准确分配到对应子订单，退费流程规范化管理
  • 用户价值：财务人员可高效管理收款、确认到账、处理待认领款项和退费
  • 成功指标：收款分账准确率 100%，退费金额校验通过率 100%

需求简述

产品架构图

      • 淘宝已付款 — 淘宝店铺收款管理，支持确认到账

需求详述

4.1 收款管理

4.1.1 状态机

常规收款状态：未核验(10)→确认到账→已支付(20)+触发分账。待认领状态：待认领(0)→认领→已认领(1)+创建收款+触发分账。

4.1.3 逻辑补充

确认到账流程：

  1. 校验收款状态为未核验(10)
  2. 更新收款状态为已支付(20)，记录到账时间
  3. 触发分账服务：按子订单ID升序，FIFO分配收款金额到各子订单
  4. 更新订单和子订单的支付状态

待认领款项认领流程：

  1. 用户选择待认领记录，输入关联的订单ID
  2. 校验订单状态为未支付(20)或部分支付(30)
  3. 校验：已有收款 + 待认领金额 ≤ 订单实收金额
  4. 自动创建一条已支付状态的收款记录(payment_collection)
  5. 触发分账生成
  6. 更新待认领记录为已认领(1)
例：财务人员通过Excel导入一笔待认领款项（微信支付，金额500元）。确认该款项对应学生张三的订单(ID=10)后，点击"认领"并输入订单ID=10。系统自动创建收款记录、按FIFO规则将500元分配到订单的各子订单，并更新订单支付状态。

4.2.2 逻辑补充

分账计算规则（FIFO）：

  1. 获取收款金额作为待分配余额
  2. 按子订单ID升序遍历所有子订单
  3. 计算每个子订单已分配金额和还需金额(实收 - 已分配)
  4. 本次分配 = MIN(待分配余额, 还需金额)
  5. 创建分账记录，更新待分配余额
  6. 余额为0时停止分配
