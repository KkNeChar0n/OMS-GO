审批流管理

需求目标

提供灵活的审批工作流引擎，支持审批类型定义、流程模板配置（多节点+会签/或签）、审批实例管理和抄送机制。为退费等业务提供标准化的审批流程支撑。

  • 业务目标：实现审批流程的标准化和可配置化，支持多级审批和灵活的审批策略
  • 用户价值：管理员可自定义审批流程模板，审批人可高效处理审批任务
  • 成功指标：审批流程流转准确率 100%，会签/或签逻辑正确率 100%

4.3 审批流管理

4.3.1 状态机

审批流状态：待审批(0)、已通过(10)、已驳回(20)、已撤销(99)。审批逐节点流转，所有节点通过则整体通过，任意节点驳回则整体驳回。

4.3.3 逻辑补充

会签节点(type=0)：

  • 通过条件：该节点所有审批人都审批且都通过
  • 驳回条件：任意一人驳回 → 删除该节点其他待审批人员 → 节点驳回 → 整个审批流驳回

或签节点(type=1)：

  • 通过条件：任意一人通过 → 删除该节点其他待审批人员 → 节点通过 → 流转下一节点
  • 驳回条件：所有人都审批且都驳回 → 节点驳回 → 整个审批流驳回

节点流转规则：

  1. 当前节点通过后，step+1，系统检查是否还有下一个节点
  2. 有下一节点：创建新的节点实例和审批人记录，继续审批
  3. 无下一节点：整个审批流标记为已通过(10)，记录完成时间，触发业务回调（如退费处理）

抄送机制：审批流创建时根据模板配置自动生成抄送记录，抄送人可在"抄送我的"标签页查看审批详情，但无审批权限。

例：创建退费审批模板，配置2个节点：节点1"财务审核"（或签，审批人：财务A和财务B），节点2"总监审批"（会签，审批人：总监C）。发起退费审批后，财务A或财务B任一人通过即可流转到节点2；节点2需要总监C通过后，整个审批流才完成，触发退费执行。
